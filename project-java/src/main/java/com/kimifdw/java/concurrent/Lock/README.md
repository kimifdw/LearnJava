# 锁
1. 死锁

1.线程饥饿。
    1. 高优先级线程吞噬所有的低优先级线程的CPU时间。
    2. 线程被永久堵塞在一个等待进入同步块的状态。
    3. 线程在等待一个本身也处于永久等待完成的对象。
2. 可重入性。
3. ReentrantLock。一个可重入的互斥锁，为lock接口的主要实现，每次最多只有一个线程持有。
    1. 创建一个非公平的锁
    2. 创建一个公平的锁。【顺序执行】。 当持有锁的时间相对较长，或者请求锁的平均时间间隔较长，那么应该使用公平锁。
    3. 定时的锁等待；
    4. 可中断的锁等待、公平性；
    5. 实现非块结构的加锁
